# RUNE Role Index

Roles for keeping RUNE neutral, extensible, traceable, and useful as Rust
contract infrastructure for AI-era systems. Read this before using any role
file.

> These roles are AI-simulated review lenses, not claims of real-person review.

## Parliament Roles

Use these roles before RUNE changes are treated as contract-ready.

| File | Voice | Primary tension |
|---|---|---|
| `parliament/contract-model-steward.md` | Contract Model Steward | Durable neutral descriptors vs. product-specific vocabulary |
| `parliament/macro-safety-steward.md` | Macro Safety Steward | Useful derivation vs. hidden generated behavior |
| `parliament/generator-interop-steward.md` | Generator Interop Steward | Extensible outputs vs. one-off schema formats |
| `parliament/vtrace-traceability-auditor.md` | VTRACE Traceability Auditor | Mission-to-evidence rigor vs. macro enthusiasm |
| `parliament/ai-contract-consumer.md` | AI Contract Consumer | Agent-readable contracts vs. source scraping and prompt convention |

## Stakeholder Roles

Use these lenses when deciding whether RUNE helps real Rust projects and
downstream systems.

| File | Stakeholder | Primary concern |
|---|---|---|
| `stakeholders/rust-maintainer.md` | Rust Maintainer | Can add RUNE incrementally without distorting idiomatic Rust |
| `stakeholders/platform-adapter-author.md` | Platform Adapter Author | Can map neutral descriptors into product-specific contracts |
| `stakeholders/future-agent.md` | Future Agent | Can inspect, validate, and safely use generated contracts |
